Purely Functional Imperative Language

PFIL is, as the name suggests, an imperative programming language that is constructed entirely of functions. Not purely functional in the traditional sense, where there are no state changes, but purely functional in that everything is a function. There are no statements. Only functions. For example, let's look at an if-then construct (without preprocessing):

if(not if(condition){
    doStuff();
}){
    doOtherStuff();
}

Now, if-then functions are, as the phrase "if-then functions" implies, functions. It's pretty obvious, honestly. However, they need to take a group of functions and execute them. Now how do they do that? The answer is that functions in this language can take lambdas (a special type that is essentially an executable block of code) as input and execute them!

PFIL's philosophy is largely similar to that of Python: batteries-included. Also, it should look cool, and whatever other Python-lovers say, curly brackets and semicolons look awesome in code. It is a fact of life.
