Primitives in the Purely Functional Imperative Language

Like all languages, PFIL is going to need some primitives.

Builtins are the basic building blocks of a language. Without them, well, you don't have a language. In PFIL, we have to sorts of primitives: builtins and importables.

Builtins are what make up the language; to call something a PFIL implementation, it must support all of the builtins. Builtins are the functions that can be used to implement almost anything that can be implemented.1I say almost because things like sockets aren't builtins, but can't be implemented with builtins. Sockets are an example of importable primitives

The builtins are as such:

  1. import(module[, name]): imports a module as name (that is, into the name namespace). If no name is given, defaults to that module's name
  2. alias(stringid, function): creates an alias. From that point on, stringid works like a function with the same properties as function
