A more technical documentation of the Purely Functional Imperative Langugage

PFIL, the Purely Functional Imperative Language, is an imperative, compiled, dynamically-typed, object-oriented, procedure-centric programming language inspired by all the best things of languages like Python and C and Haskell.1You know it's weird when it's inspired by 3 totally different languages of opposing paradigms

It is a semicolon/curly bracket language, as semicolons and curly brackets look good in code. Many may not like this, and as such a preprocessor should be made available (though this is certainly not required, as many will likely be made available by third parties) that will use pythonic syntax (Colons and indent to denote a block of code, newline (unless enclosed in certain delimiters) to denote end-of-line).
